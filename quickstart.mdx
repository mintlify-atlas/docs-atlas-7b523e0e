description: "Get NLog up and running in your .NET application in minutes."
---

<Steps>
  <Step title="Install NLog">
    Add the NLog NuGet package to your project:

    ```bash
    dotnet add package NLog
    ```

    Or via the NuGet Package Manager Console in Visual Studio:

    ```powershell
    Install-Package NLog
    ```
  </Step>

  <Step title="Create NLog.config">
    Add an `NLog.config` file to your project root. This example configures a console target and a file target:

    ```xml NLog.config
    <?xml version="1.0" encoding="utf-8" ?>
    <nlog xmlns="http://www.nlog-project.org/schemas/NLog.xsd"
          xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
          autoReload="true">

      <targets>
        <target xsi:type="Console" name="console"
                layout="${longdate} ${level:uppercase=true} ${logger} ${message} ${exception:format=tostring}" />

        <target xsi:type="File" name="file"
                fileName="${basedir}/logs/app.log"
                layout="${longdate} ${level:uppercase=true} ${logger} ${message} ${exception:format=tostring}" />
      </targets>

      <rules>
        <!-- Log everything from Debug and above to both targets -->
        <logger name="*" minlevel="Debug" writeTo="console,file" />
      </rules>

    </nlog>
    ```

    <Note>
      Set the file's **Copy to Output Directory** property to `Copy if newer` so it is present next to your compiled binary at runtime.
    </Note>
  </Step>

  <Step title="Get a logger">
    Obtain a logger instance using `LogManager.GetCurrentClassLogger()`. Call this once per class and store it in a static field:

    ```csharp
    using NLog;

    public class OrderService
    {
        private static readonly Logger Logger = LogManager.GetCurrentClassLogger();
    }
    ```

    `GetCurrentClassLogger()` captures a stack trace to name the logger after the calling class (namespace + class name). Avoid calling it inside loops or hot paths. For a named logger use `LogManager.GetLogger("MyLogger")` instead.
  </Step>

  <Step title="Log at various levels">
    NLog defines six log levels ordered by severity:

    | Level   | Ordinal | When to use |
    |---------|---------|-------------|
    | `Trace` | 0       | Most verbose. Development diagnostics, rarely enabled in production. |
    | `Debug` | 1       | Internal application events useful when debugging. |
    | `Info`  | 2       | Progress or application lifetime events. |
    | `Warn`  | 3       | Recoverable issues, validation failures, or temporary errors. |
    | `Error` | 4       | Failures or caught exceptions. |
    | `Fatal` | 5       | Critical failures — application is about to abort. |

    ```csharp
    using NLog;

    public class OrderService
    {
        private static readonly Logger Logger = LogManager.GetCurrentClassLogger();

        public void PlaceOrder(int orderId, string product, int quantity)
        {
            Logger.Trace("Entering PlaceOrder method");
            Logger.Debug("PlaceOrder called with orderId={OrderId}", orderId);
            Logger.Info("Order {OrderId} placed for {Quantity}x {Product}", orderId, quantity, product);

            if (quantity > 1000)
                Logger.Warn("Large quantity {Quantity} requested for order {OrderId}", quantity, orderId);

            try
            {
                // ... order processing ...
            }
            catch (Exception ex)
            {
                Logger.Error(ex, "Failed to process order {OrderId}", orderId);
                throw;
            }
        }
    }
    ```
  </Step>

  <Step title="Use structured logging">
    NLog supports structured logging with message templates. Use named placeholders in curly braces — NLog captures both the rendered string and the raw property values:

    ```csharp
    // Properties captured as structured data: UserId=42, Action="login", Success=true
    Logger.Info("User {UserId} performed {Action} with result {Success}", 42, "login", true);

    // Numeric properties can be filtered on by compatible targets
    Logger.Warn("Payment {PaymentId} took {ElapsedMs}ms — exceeds threshold",
        paymentId, elapsed.TotalMilliseconds);
    ```

    <Tip>
      Targets that understand structured data (such as a JSON-formatted file target or a log management platform like Seq) index individual property values, enabling queries like `ElapsedMs > 500`.
    </Tip>
  </Step>
</Steps>
